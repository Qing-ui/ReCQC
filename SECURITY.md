# Security policy

ReCQC 1.0, ReCQC 2.0, and the Hungarian implementation are maintained as formal
projects. Reports must identify the affected version and release tag.

Do not open a public issue for a suspected security vulnerability. Use GitHub's
private vulnerability reporting or a private security advisory for this
repository. Include the affected version, operating system, reproduction steps,
and any potentially exposed data. Do not attach confidential research datasets.
For private coordination, contact `2638614209@qq.com`.
